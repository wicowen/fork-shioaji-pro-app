# 台指期貨策略開發 — 研究結論與開發藍圖

> 對象：人類閱讀，幫助理解與進入狀況。
> 狀態：研究討論已收斂，尚未實作。實作用的技術規格（英文版，供 blank
> session 使用）：`2026-0702-2353-futures-strategy-research-claude-fable-5-en.md`。

## 1. 這次討論的結論（TL;DR）

- 額度微台（TMF）× 3 口。**成本結構直接否決「純搶短 scalp」**：來回固定
  成本 5–8 點，搶 3–5 點是結構性負期望。可行的形狀只有兩種 —— 單筆目標
  15 點以上，或至少單邊限價被動進場。
- 策略引擎確定採 **Python 獨立進程（daemon）**，不放前端。現有 app 的觸價
  單、括號單、風控都跑在瀏覽器分頁裡，頁面一關就死 —— 只能當監控駕駛艙。
- 推薦組合：**趨勢日捕捉 + Order-flow 確認突破 雙軌回測**，用 ORB（開盤
  區間突破）先驗證回測管線正確性。
- 無論選哪個策略，**資料錄製要立刻開始**：Shioaji API 沒有歷史五檔資料，
  今天不錄，永遠沒有。
- 這個額度的誠實天花板：做得好一年約 10–15 萬台幣。真正的目標是產出一份
  可信的實盤績效紀錄，證明系統為正期望後升小台（成本占比反而更低）。

## 2. 成本數學（一切判斷的地基）

以台指約 28,000 點估算，微台 1 點 = NT$10，契約值約 28 萬：

| 項目 | 金額 | 換算點數 |
|---|---|---|
| 期交稅（雙邊，各 2/100,000） | 約 NT$11 | 1.1 點 |
| 手續費（雙邊，行情 NT$15–25/邊） | NT$30–50 | 3–5 點 |
| 市價單吃 spread（雙邊） | NT$10–20 | 1–2 點 |
| **每回合合計** | **NT$50–80** | **5–8 點** |

搶 3–5 點的 scalp 需要 60% 以上勝率才打平 —— 手動嚴選機會可能做得到，
程式會把邊際訊號也執行，一天 20 筆就是每天固定流失約 NT$1,000。

## 3. Edge 從哪來（策略想法的檢驗框架）

散戶對機構，速度、資訊、模型三條路全是死路。真實存在的 edge 只有四種，
任何策略先回答「我賺的是誰的錢」：

1. **行為財務** —— 賺別人恐慌與追價的錢（趨勢日 = 搭 herding 便車）
2. **結構性** —— 賺制度的錢（結算機制、造市商避險流）
3. **時間維度套利** —— 機構嫌太小不撿的機會（微台規模反而是資格）
4. **執行紀律** —— 自動化的真正價值不是找新 edge，是保護既有 edge 不被
   人性磨掉（scalper 最典型的失敗是趨勢單抱不住）

說不出來源的策略，回測再漂亮都是曲線擬合。

## 4. 手上已有的武器（專案盤點）

- **即時資料**：單一 SSE 串流 tick（含 `tick_type` 主動買賣別）+ 五檔
  （含 `diff_vol` 增減量），斷線自癒。Tick 還自帶 `underlying_price`
  （現貨指數），基差是免費的。
- **訊號引擎**：`src/lib/order-flow.ts` 的 OrderFlowEngine 已量化 CVD、
  滾動買賣壓、大單衝擊 —— 盤口力道的判斷已經是程式碼，回測時移植成
  Python 即可。
- **執行鏈路**：期貨下單/改單/刪單、括號單 OCO、風控 kill-switch 都在，
  但全是前端的 —— 實盤時 daemon 要自己擁有這些邏輯。
- **歷史資料**：tick 級自 2020-03（連續合約 TXFR1/TMFR1，含夜盤、含
  內外盤別）—— 回測可行。K 棒單次查詢約 30 天上限。
- **缺口**：沒有歷史五檔（盤口類訊號無從回測）、沒有美股行情（夜盤策略
  只能靠台指自身反應）、觸價單只是客戶端模擬。

## 6. 兩個核心策略的規則雛形

### 策略 1：趨勢日捕捉

台指一年約 15–20% 交易日是單邊趨勢日。判定（四條件 AND，皆為待回測驗證
的先驗）：跳空不回補、首小時收在區間極端 20%、CVD 與價格同向無背離、
首小時波幅大於 20 日均值 1.5 倍。進場**不追突破、等第一次淺回檔**
（趨勢日回檔典型不過 33%），移動停損或 13:25 收盤出場，停損 15–20 點，
3 口全用（頻率低）。

心理提醒：勝率低、靠尾部大贏，會連吃 5–8 次小停損才等到一次 100–300 點
—— 與 scalper 的習慣**完全相反**，這是最大的執行風險。

回測要先回答的生死題：2022 年夜盤拉長到 14 小時之後，日盤趨勢日還剩
多少？若占比掉到 10% 以下，改夜盤版或放棄。

### 策略 2：Order-flow 確認突破

**用大台（TXF）的訊號，下微台（TMF）的單** —— 微台五檔基本是造市商照著
大台調的鏡像報價，自身沒有資訊含量。

```
TXF tick 流 ──> burst 大單衝擊偵測 ──┐
TXF tick 流 ──> 30s 滾動買賣壓 >65% ──┼──> 三重 AND ──> 進場 TMF
TXF 五檔   ──> 對手牆行為分析 ────────┘        （關鍵價位處）
```

關鍵價位：前日高低、日盤開盤價、夜盤高低、整數關卡、開盤區間邊緣。
牆行為的核心區別：掛單減少**且成交量大** = 牆被吃穿（真突破）；掛單消失
**但沒什麼成交** = 抽單（常是誘多誘空，不做或反著想）。目標 15–30 點，
停損掛結構位（回破突破點即失效）、上限封頂 12 點，先 1 口。

殘酷事實：造市商看大台比我們快，訊號到下單的鏈路是百毫秒級 —— 回測
成交價必須用「訊號後 500ms 的對手價」，否則期望值虛胖 1–2 點/邊。

## 7. 回測會騙人的五個地方

1. **試撮污染** —— 8:30–8:45、13:40–13:45 的試撮價量不可成交；先驗證
   歷史 ticks 是否含試撮再寫開盤邏輯
2. **滑價假設** —— 上述 500ms 規則
3. **分段驗證** —— 2020–2022 與 2023 之後分開看（夜盤改制、微台上市）；
   全樣本漂亮但近兩年翻負 = 已死的策略
4. **成本敏感度** —— 成本從 5 點掃到 9 點畫期望值曲線，+2 點就翻負的
   策略太脆弱
5. **樣本外紀律** —— 最後 12 個月鎖起來，參數凍結後只看一次，看過就
   不能再調，違反即整份回測作廢

## 8. 開發路徑

```
Phase 0 資料錄製(立即)      Phase 1 離線回測           Phase 2 影子交易(2-4週)     Phase 3 微量實盤
+--------------------+     +--------------------+     +--------------------+     +--------------------+
| Python 錄製 daemon |     | 歷史 tick 下載器    |     | 策略 daemon 上線   |     | 1 口 -> 3 口       |
| TXF+TMF            | --> | (TXFR1 2020~)      | --> | 只發訊號不下單     | --> | daemon 持有風控:   |
| Tick + 五檔        |     | 回測引擎(含七節規則)|     | app 加訊號面板     |     |  單筆停損/日虧 3R  |
| -> parquet 日檔    |     | ORB 先驗證管線     |     | 與回測邏輯比對一致 |     |  強制平倉+鎖單     |
| 缺口監測/斷線重連  |     | 趨勢日+突破雙軌    |     | 手動確認下單過渡   |     | 連虧2日停機檢討    |
+--------------------+     +--------------------+     +--------------------+     +--------------------+
        |
        +--> 同時進行: 手續費去談(最高 ROI 的一件事) + 手動交易影子日誌(比對人 vs 規則)
```

驗收門檻（過不了就不進下一階段）：

- **Phase 0**：盤中無 >5 秒的靜默缺口（含 08:22 維護重訂閱）、simtrade
  與 diff_vol 欄位完整保留、無人值守跑一週不斷
- **Phase 1**：兩個分段都扣 7 點成本後期望值 > 0、+2 點壓力測試不翻負、
  樣本數達標（趨勢日 >=60 筆、ORB >=300 筆）、最大回撤 <=20 個停損單位、
  樣本外一次性驗證同號
- **Phase 2**：tick 到訊號延遲 <1 秒、live 訊號與回測邏輯在同一天資料上
  重放結果一致
- **Phase 3**：風控完全由 daemon 持有，與 UI 無關

## 9. 資金與風控框架

- 建議帳戶 NT$60,000–80,000：保證金約 4 萬 + 回撤緩衝（低於 15 個停損
  單位 = 資金不足）
- 鐵律：單筆風險 <= 帳戶 1.5%、日虧上限 3 個停損單位（daemon 強制平倉
  + 鎖單）、連虧 2 日停機檢討

## 10. 待決事項（會改變優先級的三個問題）

1. **可交易時段**：日盤 / 夜盤 / 全自動不盯盤？夜盤為主則策略 4 升核心、
   全自動則風控工程量大增
2. **實際手續費**：>NT$20 短線全出局；NT$10–15 策略 2 可評估。
   還沒談的話，談手續費是所有事情裡投資報酬率最高的一件
3. **最信任的盤口訊號**：大單掃盤延續 / 牆被吃穿 / 抽單反向？
   決定策略 2 的規則長相；抽單類最難量化，建議記錄復盤而非自動化
