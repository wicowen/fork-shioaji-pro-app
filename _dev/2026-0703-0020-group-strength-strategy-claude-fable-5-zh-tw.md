## 1. 想做什麼

1. 各族群排列強弱（這個 app 原本就有雛形）。
2. 把族群內連動性高的成員抓出來組成 group。
3. 針對 group：出現「強轉弱」時，放空群內最弱的那一檔（看A空B）。

## 2. 現況掃描：已經有什麼、缺什麼

### 已有（可直接復用）

| 功能 | 位置 | 說明 |
|------|------|------|
| 類股強弱快照 | `components/sector-heatmap.tsx` | 26 個 TWSE 類股指數依漲跌 % 排序（總覽），可下鑽成分股；20 秒輪詢 snapshot |
| 產業分類資料 | `lib/stock-index.ts` | 全市場個股的 `category`（TWSE 28 分類）＋類股指數對照表＋名稱搜尋 |
| 排行榜 | `components/scanner-panel.tsx` | 漲/跌/量/額排行，已有「放空」複選模式（跌幅＋量＋額交集） |
| 盤口力道引擎 | `lib/order-flow.ts` | 單一商品的 CVD、滾動主動買賣比（buyRatio）、大單 burst 偵測；無框架依賴、可多實例 |
| 警示型監控引擎模板 | `lib/rollover-engine.ts` | 只發警示不下單、含 hysteresis 防抖（觸發後須回落才重新武裝）——新引擎的最佳模板 |
| 觸價引擎 | `lib/trigger-engine.ts` | 固定價格穿越 → 市價單，含 OCO |

| 下單 API | `lib/shioaji.ts`、`lib/trade.ts` | 股票/期貨/組合單；`fetchKbars`（單次約 30 天）、歷史 tick |

### 缺（要新建）

1. 強弱「時序」：熱力圖只有快照，沒有排名隨時間變化，無法定義「轉」。
2. 個股間相關性計算：全 repo 沒有任何相關性/z-score/beta 數學。
3. 族群層級 order-flow 聚合：現有引擎是單商品的。
4. 條件式訊號引擎：觸價引擎只會比對固定價格。
5. 訂閱數防護：Shioaji 上限約 200，且本 app 每檔訂 Tick＋BidAsk 吃 2 個訂閱（實際天花板約 100 檔），目前完全沒有管理。

### 兩個關鍵陷阱（掃描發現）

- **自動化路徑目前送不出空單**：`daytrade_short`（現沖先賣）只接在手動下單票；自動引擎共用的 `trade.ts sendOrder()` 送股票單不帶它，也沒有融券欄位。要自動空現股必須先補這段。個股期貨走 `placeFuturesOrder` 則沒有此問題。
- **所有引擎都是客戶端**：App 關掉、分頁休眠，觸價/風控/訊號全部停止。全自動策略掛在這上面有結構性風險。

## 3. 策略評估（討論核心）

### 成立的部分

- 台股族群連動與 lead-lag（資訊在族群內傳導的時間差）真實存在：題材資金輪動、隔日沖主力都以族群為單位進出。
- 「空最弱」方向正確：族群轉弱時弱者缺乏買盤保護、補跌最快；空龍頭反而容易遇到主力防守被軋。
- 本質：交易「傳導時間差＋弱者下跌 beta 的不對稱」。

### 三個決定成敗的問題（都可量測，不要拍腦袋）

1. **可交易時間窗有多長**——刻意挑連動性高的成員，代表傳導快；等訊號確認時弱者可能已經跌完，策略退化成「追空已跌的股票」。必須用分 K 做 cross-correlation 量測 lead-lag 有幾根 bar。
2. **「強轉弱」怎麼定義**——太敏感被 whipsaw 巴、太遲鈍行情走完。建議差異化：不要只用價格排名（落後、人人看得到），把盤口力道推廣到族群層級，「價格未跌、主動賣壓先湧現」是更早的領先訊號。
3. **空弱者的執行成本**——最弱的股票常伴隨流動性差、spread 寬、券源少、處置股。現沖稅費 0.15%＋滑價會吃掉薄 edge。建議個股期貨優先（免券源、無平盤下限制、稅低），代價是只有 200 多檔標的（反而強迫聚焦流動性）。

### 其他風險

- 相關性是 regime-dependent：趨勢日連動、盤整日解耦、恐慌日全市場相關性趨近 1。分組要 rolling 重算＋盤性濾網。
- 軋空：弱者浮額輕易被反拉；停損必須硬性，並接受漲停鎖死時停損失效的尾部風險。
- 策略型態：「純空弱者」符合族群補跌假設；「空弱買強」的 pair 在族群整體轉弱時 long 腿會吃掉獲利，不建議當主形態。

### 設計歧義（已先採預設）

「群內每一個標的出現強轉弱就空最弱」的訊號源有三種解讀：
(a) 族群聚合轉弱（預設，雜訊最低）、(b) 指定龍頭轉弱、(c) 任一成員轉弱（假訊號最多）。

## 4. 訊號資料流（Phase 1 設計）

```
 SSE tick (單一連線)
      |
      v
 stream.ts onAnyTick
      |
      v
 group-strength engine（新建，仿 rollover-engine）
      |-- 每成員一個 OrderFlowEngine ──> buyRatio / CVD 斜率
      |-- 強弱分數 = z(當日漲跌%) + z(buyRatio-0.5)   （每秒級重算）
      |-- 排名時序：每 10s 快照 rank（ring buffer）
      |-- 族群聚合：成員分數中位數 + 上漲廣度
      |
      v
 強轉弱規則（含 hysteresis，一個事件只警示一次）
      |
      v
 notify() ──> 通知中心 + 面板事件 log      ※ Phase 1 不下單
```

分組驗證（離線、非即時）：

```
 fetchKbars 1分K（近20交易日）──> 聚成 5 分報酬 ──> pearson 相關矩陣
                                                   └─> 群組編輯器顯示成員兩兩相關係數
```
