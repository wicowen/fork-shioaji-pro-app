## v0.1.11 — Port 衝突自癒與正式環境憑證

### 8080 被占用也能啟動
先前 8080 port 被占用時內建伺服器會啟動失敗、畫面一直 LOST。現在啟動流程會自動處理：

- 偵測到**既有的 shioaji server**（例如你自己用 CLI 啟動的）→ 直接連接使用，不再互搶 port
- 8080 被**其他程式**占用 → 自動改用下一個可用 port（8081…）啟動，畫面自動跟隨新 port 重新載入
- App 重開時若伺服器還在運行（含非預設 port），自動同步連回

### 正式環境憑證（CA）
正式環境下單需要 Sinopac.pfx 憑證，現在可以直接在介面設定：

- 伺服器管理新增**憑證區**：原生檔案選擇器挑選 `.pfx`、輸入憑證密碼（儲存在本機 App 資料夾）
- 啟動伺服器時自動帶入憑證設定，登入時自動啟用
- 選擇正式環境但未設定憑證時會擋下並提示（憑證請至 sinotrade.com.tw 的 API 管理頁下載；模擬環境不需要憑證）
