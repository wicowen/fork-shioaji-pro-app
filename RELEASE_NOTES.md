## v0.1.20 — 修正卡在「啟動中」、憑證問題一眼看穿

### 伺服器卡在「啟動中」修復
- 根因：`shioaji server start` 以前景執行、永不退出，App 卻等它跑完 → 永遠等不到 → UI 卡在「啟動中」、按鈕全部 disable（伺服器其實早就起來了）
- 改用 spawn＋健康輪詢：伺服器真的回應才返回；並補上對應權限（`shell:allow-spawn`）
- 防呆：有 LIVE 行情就不再顯示「啟動中」；busy 安全逾時，按鈕不會卡死

### 正式環境憑證（CA）問題一眼看穿
- 伺服器即使 CA 啟用失敗仍會照常啟動（只在 log 留一行），以前要到下單 400 才發現。現在啟動後直接在憑證區紅字顯示原因：
  - **憑證已過期** → 請至 API 管理頁重新下載 Sinopac.pfx
  - **憑證密碼錯誤** → 請確認下載憑證時設定的密碼
- 憑證密碼欄位加上**顯示/隱藏切換**（眼睛圖示），方便核對密碼有沒有打對
- 搭配 v0.1.19 的「下單就緒檢查」與錯誤訊息透明化，正式環境下單 400 的原因現在都看得到
